Home / Frontend

Applications

  • patient-app: booking, consult-today, notifications, PWA; translations (en/ml).
  • nurse-app: queue dashboard, token calling, break scheduling.
  • clinic-admin: admin dashboards, onboarding, departments, doctors, reports.
  • superadmin: system-level management (not deployed by default).

Component Hierarchy

  • Shared UI: `packages/shared-ui/src/components/PatientForm.tsx` used in patient/nurse flows.
  • Clinic admin UI kit under `apps/clinic-admin/src/components/ui/*` (Radix wrappers).
  • Dashboard widgets (charts/tables) under `apps/clinic-admin/src/components/dashboard/*`.

State Management & Hooks

  • Local state + React hooks; SWR in patient-app for data fetching.
  • Custom hooks: `useAppointmentStatusUpdater`, `useDoctorStatusUpdater`, `useQueueManagement` handle real-time Firestore updates.
  • Auth context per app (Firebase client). Role-based gating in layouts.

Forms & Validation

  • React Hook Form + Zod; see clinic-admin appointments page schema (name, phone, age, bookedVia rules).
  • Error handling with `FormMessage`; input components in `components/ui` follow Radix patterns.

Routing & Navigation

  • Next.js App Router with grouped routes: `(public)` for login/signup, `(app)` for authenticated areas.
  • Breadcrumbs and dashboards per app; API routes colocated in `app/api/*`.

Performance & UX

  • Dynamic imports for heavy forms (patient consult-today).
  • PWA setup in patient/nurse for offline caching; manifest assets provided.
  • Charts and tables paginate/filter; skeleton loaders and toasts for feedback.
  • Accessibility: Radix primitives, semantic inputs; continue WCAG checks for forms and contrasts.
Reuse shared-ui and shared-core to avoid divergence in booking flows across apps.
