Deployment Process (Vercel)

  1. Create 3 Vercel projects with root directories: patient-app, nurse-app, clinic-admin. (superadmin optional).
  2. Add env vars (all `NEXT_PUBLIC_FIREBASE_*`, Twilio keys where needed, `GOOGLE_SERVICE_ACCOUNT_JSON` for uploads).
  3. Run `pnpm build` locally to verify.
  4. Push to main; Vercel builds per project.
  5. Test preview; then promote to production domains: app.kloqo.com, nurse.kloqo.com, clinic.kloqo.com.

Environments

  • Dev: `.env.local` in each app (ignored). Use test Firebase project.
  • Prod: Vercel env settings; same keys for build/runtime.
  • Feature flags: none formal; use env toggles (e.g., `NEXT_PUBLIC_DEBUG_BOOKING`).

CI/CD

  • Recommended GitHub Actions (see NEXT_STEPS.md snippet): install, test, typecheck, lint, build.
  • Turbo caching for builds; set `globalDependencies` = `**/.env.*local` (see turbo.json).

Infrastructure Notes

  • Firebase project: Firestore, Auth, Storage, FCM. Ensure Firestore rules locked down.
  • Storage bucket: `kloqo-clinic-multi-33968-4c50b` referenced in upload route.
  • SMS: Twilio credentials needed for production.

Monitoring & DR

  • Logging currently console-based; add Sentry/LogRocket for prod.
  • Backups: use Firebase export for Firestore; version scripts for reference data seeds.
  • Uptime: add external monitors (Pingdom/UptimeRobot).
Before production, run the checklists in NEXT_STEPS.md (envs, rules, builds, tests).
